# 每份 md 的统一结构（强约束）
1. **🎯 学习目标**：读完能回答的 3-5 个问题
2. **🗂️ 源码定位**：精确到文件路径、关键类/函数、必要时给行号区间
3. **🧭 架构/流程图**：必须用 Mermaid 绘制（流程图、时序图或类图）
4. **🔍 核心逻辑讲解**：
    - 贴关键代码片段
    - 逐块解读，重点说 **"为什么这么设计"**、**"换种方式会有什么问题"**
    - 深入拆解其核心函数的逻辑执行伪代码或关键源码片段，讲解状态机（State）是如何流转的。
    - 阐述设计该模块时，为了解决 Agent 的什么痛点（例如：为什么这里不用传统的 Chain 而是用 LangGraph 的特定 StateGraph 节点）。
    - 至少列出 1 个该模块的设计权衡（trade-off）
5. **🧩 体现的通用 Agent 设计模式**：本模块对应哪些可复用模式（如 ReAct / Plan-and-Execute / Reflection / Supervisor 等）
6. 与 Agent Harness 六要素（反馈循环 / 记忆持久化 / 动态上下文 / 安全护栏 / 工具集成 / 可观测性）的对应关系
6. **⚠️ 常见坑与调试技巧**：列出 2-4 个真实可能踩的坑
7. **🛠️ 动手实操**（如该模块可实操，必须包含）：
    - **最小可运行示例**：脱离原项目复杂度，给出 30-80 行独立可跑的代码，复现该模块核心机制
    - **调试任务**：明确说明在哪打断点 / 打印什么 state / 如何人为制造异常观察行为
    - **改造练习**：1-2 个动手改造题（如替换工具、加一个新节点、改 prompt 看输出差异）
    - **预期输出 & 验证方式**：我怎么知道我做对了
8. **🎤 面试视角**：
    - 结合该模块，提炼出 2-3 个在高级 Agent 岗位面试中极具深度的问题（例如：“如何解决 Sub-agent 之间并发操作导致的主状态冲突？”、“沙箱执行超时或死循环时，宿主机如何进行安全的信号熔断？”）。
    - 提供教科书级的标准答案与工程实践最优解。
9. **📚 延伸阅读**：相关论文、官方文档、博客链接
