# 文档拆分原则（按实际代码调整）
- **项目总览与环境搭建**：项目定位、目录结构地图、依赖关系、本地启动 / Docker 启动步骤、最小可跑通的 Hello World。
- **LangGraph 图与状态流转**：全局 State 设计、节点/边的定义、条件路由、Human-in-the-loop、异常 Fallback。
- **子智能体编排（Sub-agents）**：角色分工（研究员/程序员/测试员等）、消息传递协议、交接棒机制、并行分支。
- **记忆系统与上下文工程**：短期/长期记忆、向量检索 RAG 链路、Token 压缩与 Offload 策略、动态 Prompt 拼接。
- **沙箱生命周期管理**：Local / Docker / K8s 三种模式的差异、代码注入与 Stdout/Stderr 回收、超时与资源隔离、Self-Correction 闭环。
- **技能库（Skills）与 MCP 工具集成**：Markdown 驱动的 Skill 加载机制、MCP 协议接入、工具调用的参数清洗与异常隔离。
- **异步并发与模型网关**：asyncio 使用模式、并发分支调度、Rate Limit 防护、多模型路由（OpenRouter / 轻重模型混用）。
- **反思纠错与结构化输出**：Traceback → Observation → Reflection 链路、最大重试次数、Pydantic / response_format / 正则兜底的 JSON 解析防御。
- **评估、可观测性与 Token 经济学**：Pipeline 打点、Benchmark 设计思路、Prompt Caching、轻量节点降本策略。

> 如果实际代码结构与上述主线不完全吻合，请以源码事实为准，重新调整切分

# 每份 Markdown 的统一结构模板
每份文档必须严格包含以下章节，缺一不可：

## 1. 模块定位（Why this matters）
- 这个模块在整体架构中扮演什么角色？
- 不读这部分会错过什么关键认知？
- 为什么 deer-flow 要这么设计？
- 与 Agent Harness 六要素（反馈循环 / 记忆持久化 / 动态上下文 / 安全护栏 / 工具集成 / 可观测性）中的哪一项对应？

## 2. 源码地图（Source Map）
- 列出本模块涉及的**所有关键文件路径**（相对仓库根目录）。
- 用一个表格列出关键类 / 函数 / 常量，每行包含：名称、所在文件:行号、一句话职责。
- 用 Mermaid 流程图 / 时序图 / 状态图（至少 1 张）画出本模块的内部结构、与其他模块的交互、状态/消息流向。**禁止只用文字描述**。

## 3. 核心逻辑精读（Deep Dive）
- 给出关键文件的相对路径（例如 `src/graph/builder.py:42`）；
- 关键代码片段要**直接引用项目源码**（带行号），不要伪代码
    - 逐行/逐块讲解：这段代码在做什么？为什么这样写？换一种写法会有什么问题？
    - 解释"为什么这么写"而不仅是"写了什么"，对比常见替代方案的取舍
    - 有哪些值得借鉴的工程技巧？
    - 标注出"作者的精妙之处"和"可能的改进空间"。
- 必须明确点出涉及的设计模式（状态机、责任链、观察者、策略等）。

## 4. 关键问题答疑（Key Questions）
针对该模块，必须正面回答以下类型的问题（按模块特性挑选 4–6 个最相关的）：
- 全局 State 的字段是怎么定义和演化的？多智能体如何共享/隔离？
- Sub-agent 之间的消息结构长什么样？
- 长上下文是如何压缩 / Offload 到文件系统的？
- 沙箱里的 stdout/stderr 是怎么捕获并回传给 LLM 的？
- 工具调用超时 / 报错时，重试链路怎么走？最大重试次数在哪里设定？
- JSON 解析失败时的 fallback 策略是什么？
- asyncio 并发分支如何防止打爆 LLM Rate Limit？
- 评估打点埋在了哪些位置？怎么读取 trace？

## 5. 横向延伸与面试级洞察（Interview-Grade Insights）
- 这个模块体现了 Context Engineering / Harness 工程 的哪个具体原则？
- 如果让我去面试高级 AI 工程师，这个模块能讲出哪 2–3 个有深度的观点？
- 业界其它项目（如 AutoGen、CrewAI、LangGraph 官方示例、OpenAI Swarm）在同样问题上的不同解法是什么？做一个简短对比。

## 6. 实操教程（Hands-on Lab）⚠️ 必须有
- **最小可运行示例**：给出一段可以直接 `python xxx.py` 跑起来的代码，复现本模块的核心机制。
- **Debug 任务清单**：列出 3 个我可以亲手做的 debug 实验，例如：
    - 在 XXX 函数打断点，观察 state 在节点切换时的变化；
    - 故意让沙箱里的 Python 抛 `ZeroDivisionError`，跟踪 Reflection 链路；
    - 把某个 Pydantic 字段改成非法值，看 retry 逻辑触发几次。
- 每个实验注明：操作步骤、预期观察到的现象、可以学到什么。

## 7. 与下一模块的衔接
- 学完本模块后，下一步该看哪份文档？为什么是这个顺序？
